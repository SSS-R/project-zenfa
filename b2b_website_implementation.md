# Zenfa AI Vendor Portal (B2B) Implementation Plan

## Overview
This platform allows PC component vendors (e.g., StarTech, Ryans, Skyland) to purchase and manage API access to the Zenfa AI Build Engine. This will enable them to integrate an intelligent AI PC builder directly into their own retail websites.

## Core Features Needed

### 1. Vendor Authentication & Onboarding
- **Sign up / Login:** Vendors can create accounts using corporate emails.
- **Business Profile:** Collect store details, website URL, and primary contact.
- **Verification:** Manual or automated verification to ensure they are legitimate businesses.

### 2. Subscription & Billing (SaaS Model)
- **Pricing Tiers:**
  - **Basic:** 500 AI evaluations/month (e.g., 2,000 BDT)
  - **Pro:** 2,000 AI evaluations/month (e.g., 5,000 BDT)
  - **Enterprise:** Unlimited/Custom limits (e.g., 20,000 BDT)
- **Payment Gateway Integration:** SSLCommerz or Stripe for recurring billing.
- **Usage Tracking:** Clean dashboard showing daily/monthly API calls remaining.

### 3. API Key Management
- **Generate/Revoke Keys:** Vendors can generate API keys for their development and production environments.
- **Security:** Ability to whitelist specific domains (CORS) or IP addresses to prevent key theft.

### 4. Developer Documentation (API Integration)
Vendors need to build their own frontend UI that collects inputs from their customers and sends it to our API.
- **Frontend Input Guidelines:** Guide vendors on how to build the input forms on their sites (e.g., collecting Budget, Purpose, Brand Preferences).
- **API Reference:** Endpoints for `/v1/build` and `/v1/compatibility/check`.
- **Integration Scripts:** "Drop-in" code snippets or SDKs in JavaScript, React, Python, and PHP to easily embed the builder into standard e-commerce platforms (WooCommerce, Shopify, custom sites).
- **Webhooks (Optional):** Alerting the vendor when they reach 90% of their API quota.

### 5. Vendor Dashboard Analytics
- Total PC builds generated by their customers.
- Most popular budgets and purposes.
- Conversion tracking (did the API lead to a cart checkout?).

## Session & Quota Management (This App's Responsibility)

> **Important:** The Zenfa AI engine is **stateless per-request** and only produces structured logs. All session state, quota tracking, and analytics storage is handled by this B2B portal backend, NOT the engine.

### What This App Manages:
- **API Quota Tracking:** Track each vendor's monthly API call count against their subscription tier. The engine returns structured logs; this portal aggregates them.
- **Key-Domain Binding:** Store the mapping of API keys to whitelisted domains in this app's database. The engine's auth middleware reads from this store (or a shared Redis cache) for domain verification.
- **Analytics Database:** Consume the engine's structured JSON logs (via a log aggregator) and store aggregated metrics in PostgreSQL/MongoDB for the vendor dashboard.
- **Billing & Invoicing:** Handle subscription lifecycle, payment processing, and invoice generation — completely separate from the engine.
